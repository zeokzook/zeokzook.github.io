CONTRIBUTIONS

Resource Area Spawning

I developed the resource and building spawning system for the game. We wanted resource areas and buildings to spawn around the player. So, I made a system where in the beginning of the game, resource areas and buildings will spawn around the player in a circle around the spaceship. On a new day, old resource areas will despawn and respawn around the spaceship. Resource Areas will not spawn close around the spaceship but will spawn outside of a certain radius that can be modified.

After we got some meshes made by our artist and the terrain of the map done, some additional changes need to be done. Since the terrain in our game is bumpy, the resource points will rotate accordingly based on the normal of the terrain. This is done by doing a line trace and getting the normals from that.

The resource points that spawn are randomly selected from a data table. This data table holds data for the mesh, the resource amount and the type. I made it into a data table so anyone can add different variants of resource areas without the need of a programmer. All that needs to be done is adding a new row and setting the new mesh and resource amount.

Building Spawning

The building spawning is also done in the same system with an added radius for the buildings to spawn further away than the max radius for the Resource Areas. These buildings give the player more things to explore and Resource Areas also spawn inside them.

Each building is a blueprint that has box collisions to determine where Resource Areas will spawn. The type and number of resource spawned in the buildings can be tweaked. Depending on the size of the buildings, we can have more Resource Areas spawn or have Resource Areas that offer more more resources.

The buildings only spawn once in the beginning of the game. It would have been great if the buildings do spawn more if you wander far away enough from the cluster of buildings so it really feels like there is more to explore. There are only two types of buildings at the moment but more could easily be added by duplicating one of the building blueprints and changing the mesh and variables depending on what we want. The only time consuming process would be adding the box components for resource area spawning.

Resource Gathering Ability

To collect resources in the game, the player has to walk up to a Resource Area and hold right click to start collecting. The player will stop collecting when the player walks too far away, stops holding right click or the resource has depleted.

This ability was easily done with the help of the Gameplay Ability System (GAS). GAS takes some time to setup and learn but it is a very useful system that I used in a project from my previous company. In the GatherResource Abiity Blueprint, all that needs to be done is to sphere trace around the player with an Ability System Component and find a Resource Area and start depleting it with a timer inside the ability. Every interval, we apply a Gameplay Effect to the Player that increases the Resource attribute.

Jetpack Ability

We also decided to add a Jetpack Ability which I was responsible for, the ability was a simple hold jump to fly, let go and you'll start to fall. This was also done in an ability utilizing the Gameplay Ability System.

This took awhile to get right and a lot of trial and error was done to make sure we gave the player enough power and airtime to get on the ship and to fly around freely with it. It is a very basic jetpack ability that allows the player to hover around and easily access areas that are quite high up (like one of our buildings). I also implented a limit on how high the jetpack can be activated. After a certain height, the jetpack will stop activating and the player will begin to fall.
